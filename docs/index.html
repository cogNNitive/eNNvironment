Cooperation

How the ecosystem co-operates

The ecosystem separates what the agent knows (iNNfo) from how the agent acts (actioNN).

Agent bootstraps

The agent fetches eNNvironment, parses the YAML frontmatter in index.html/README.md, and discovers available skills and workflows.

Skills installed

Each skill is downloaded from actioNN, validated, and registered. MCP bundles are fetched from iNNfo and wired into opencode.json.

Workflows ready

The agent presents a menu of workflows. Choose one — the agent already has the domain knowledge and tools to execute it.

AI Agent → eNNvironment (manifest + bootstrap) → actioNN (agent skill instructions) → iNNfo (innfo-mcp + specs) → Models, documents, workflows

2

Write specs or processes in Markdown files ending in _NN.md. Define structure and metadata in standard YAML frontmatter.

Open the folder in the Modeler app or run an AI agent. The core engine parses and validates all templates instantly.

FAQ

Frequently Asked Questions

Everything you need to know about cogNNitive and how it compares to other tools.

How does cogNNitive compare to Obsidian?

Obsidian is a personal note-taking tool designed for individuals. While you can open and edit iNNfo's Markdown files in Obsidian, Obsidian itself does not enforce rules, validate structural consistency, or have a native protocol for AI agents to write back changes programmatically. cogNNitive is designed for collaborative human-AI teams, adding validation, templates, and agent connectivity to standard Markdown folders.

How does cogNNitive compare to Notion?

Notion is a centralized, proprietary database wiki. Your data is locked in a vendor's database, and integrating custom AI agents requires complex API setups and expensive fees. cogNNitive is decentralized and file-based. Your knowledge catalog lives in your Git repository under absolute ownership. AI agents can access it locally via MCP, and changes are committed directly to version control.

Do I need to use the iNNfo Modeler?

No. Since it is plain Markdown, you can write and edit models in VS Code, Obsidian, Vim, or any editor you prefer. The iNNfo Modeler is a browser-based utility that parses your workspace folder locally to give you instant validation feedback, graph visualizations, and tabular matrix views.

What is the benefit of the Spec Chain?

The spec chain ensures that your documents follow formal business rules. If a team member or AI agent writes a model that doesn't fit the template (e.g., missing a required field or referencing a non-existent role), the validator catches it immediately. This ensures your knowledge base never becomes corrupted or outdated.

The Problem

Documentation is a black box for AI

Traditional tools were built for human eyes only. AI agents cannot reliably extract structured knowledge — they hallucinate structure, miss cross-references, and cannot validate correctness.

Documents Are a Black Box for AI

Notion, Confluence, wikis, and plain Markdown are designed for human eyes. AI agents cannot reliably extract structured knowledge.

Our solution: iNNfo documents are semantically structured with deterministic MCP tools — no hallucination, no guesswork.

Vendor Lock-In & Proprietary Formats

Most tools use proprietary databases, APIs, and rate limits. Your knowledge is trapped behind a vendor's wall.

Our solution: Plain Markdown files with YAML frontmatter — no proprietary database, no vendor lock-in, Git-tracked and portable.

Documentation Rots

Maintaining docs is manual, tedious, and unrewarding. Without validation, outdated documents look identical to current ones. Teams stop trusting them.

Our solution: AI agents proactively validate docs against templates, flag inconsistencies, and enforce structural integrity.

AI Workflows Are Ad-Hoc

Teams cobble together prompts and scripts for each task. Every session starts from scratch. Knowledge about how to instruct the agent is lost.

Our solution: actioNN provides versioned, reusable skills (SKILL.md files) that teach agents specific tasks — install once, use anywhere.

Knowledge Tools Don't Talk to Each Other

Documentation tools, project management, AI agents, and CI/CD pipelines operate in separate silos. No shared format or protocol.

Our solution: OKF-compatible documents work with any OKF-compliant tool. MCP provides a standard interface for AI agents.

Features

The cogNNitive ecosystem

A modular stack of tools and specifications that make knowledge AI-native — from schema to agent interaction.

S

The iNNfo Specification Stack

A four-level specification hierarchy that structures knowledge from meta-specification (Level 0) through concrete data instances (Level 3). Each level pins an immutable URL for reproducible validation.

Any document is structurally guaranteed valid — engine-backed, never LLM-dependent.
M

The innfo-mcp Server

A deterministic Model Context Protocol server exposing 7 tools for AI agents. Wraps the core parser and validator. AI agents call these tools instead of trying to parse iNNfo files themselves.

AI agents do NOT need to understand iNNfo syntax — they call deterministic tools with guaranteed correct results.
A

The actioNN Skill Collection

A curated collection of 11+ modular, versioned skills that teach AI agents how to perform specific tasks. Each is a SKILL.md file with YAML frontmatter defining its trigger and instructions.

Skills are reusable across projects, versioned for reproducibility, and self-documenting. No more prompt crafting.
E

The iNNfo Modeler

A browser-based workspace editor for iNNfo models built with Vue 3. Opens folders via the File System Access API, parses into a normalized node graph, and provides multiple editing views: Tree, Block Feed, Table, and Graph.

What validates in the editor validates everywhere — instant feedback without a server.
T

The traNNsform Pipeline

An agent-driven document import/export pipeline. Ingests PDF, DOCX, XLSX, TXT, CSV, JSON, and Markdown — normalizes, aligns to iNNfo templates, and generates validated model drafts with source traceability.

Convert years of accumulated documentation into validated, AI-readable knowledge models.

Comparison

How cogNNitive stacks up

Honest comparisons with the tools you already know.

Alternative

vs Obsidian

Obsidian has no schema enforcement, no validation engine, and no native AI-agent protocol. Plugins add capabilities but there is no standardized way for AI agents to read, validate, and write structured knowledge.
iNNfo provides deterministic validation, a 4-level spec chain, template enforcement, and an MCP-native protocol — engine-backed tools, not prompt-dependent plugins.
Alternative

vs Notion / Confluence

Proprietary databases with vendor lock-in, API rate limits, no offline access, no Git integration, and costly per-seat pricing.
File-based Markdown in Git — no lock-in, no API fees, no rate limits. Any AI agent with filesystem access can read and write. Free and open source.
Alternative

vs Plain Markdown

No schema enforcement, no validation, no structural markers, no standardized AI interaction protocol. Every team invents its own conventions, which AI agents cannot reliably parse.
Structured YAML frontmatter, typed concepts and fields, template enforcement, spec chain validation, and MCP tools for AI agents.
Alternative

vs Google OKF (vanilla)

OKF defines a file-based knowledge format but provides no validation engine, no template system, no marker syntax, no matrix support, and no MCP server for AI interaction.
Every iNNfo document is valid OKF. But iNNfo adds templates, markers, matrices, spec chain, deterministic validation, and MCP tools.

For Developers — Platform & Architecture
Platform

How the ecosystem co-operates

The ecosystem separates what the agent knows (iNNfo core engine) from how the agent acts (actioNN skills). This division keeps concerns clean and composable.

AI Agent → eNNvironment (manifest + bootstrap) → actioNN (agent skill instructions) → iNNfo (innfo-mcp + specs) → Models, documents, workflows

Agent bootstraps

The agent fetches eNNvironment, parses the YAML frontmatter, and discovers available skills and workflows.

Skills installed

Each skill is downloaded from actioNN, validated, and registered. MCP bundles are fetched from iNNfo and wired into opencode.json.

Workflows ready

The agent presents a menu of workflows. Choose one — the agent already has the domain knowledge and tools to execute it.
